## 📌 전체 프로젝트 Phase 요약

* **Phase 1: 시뮬레이션 환경 및 데이터 파이프라인 구축 (완료 및 수정)**
  * SUMO 네트워크 구성, 차량 경로 및 충전소 배치
  * 시뮬레이션 결과(XML) → 피처 텐서(NPY) 변환 파서 개발
* **Phase 2: 대규모 배치 시뮬레이션 (진행 중 - 파라미터 갱신 후 재수행)**
  * 다양한 시나리오 파라미터(용량, 충전 임계값 등) 조합 실행
  * 데이터 정제 및 모델 입력용 텐서 구축
* **Phase 3: Graph WaveNet 모델 학습 및 평가 (대기 중)**
  * 생성된 합성 데이터를 활용한 시공간 충전 수요 예측 모델 학습
* **Phase 4: 베이스라인 모델 비교 및 아키텍처 해석 (예정)**
  * LSTM, GRU 등 베이스라인과의 성능 비교
  * 학습된 Adjacency Matrix 시각화 및 해석
* **Phase 5: 차량 단위(미시적) 적용 확장 (예정)**
  * BMW i3 실데이터 기반 Graph WaveNet 동일 구조 적용 (에너지/SoH 추정)

## 🛠 기존 데이터 파이프라인 문제점 (Phase 1 & 2)

기존 설정 오류로 인해 딥러닝 학습에 부적합한 데이터가 생성됨.

* **비현실적 배터리 스펙**
  * **원인:** 차량 배터리 용량이 3~8 kWh (자전거/초소형 수준)로 설정됨.
  * **결과:** 차량당 평균 충전 시간이 80초 남짓에 불과, 병목/대기 수요 패턴 부재.
* **도착 시 SOC(%) 계산 오류**
  * **원인:** 파서 로직에 최대 배터리 용량 35,000 Wh가 하드코딩됨.
  * **결과:** 작은 용량의 배터리로 충전한 에너지를 35,000으로 나누어, **도착 시 SOC가 95%**로 기록되는 논리적 모순 발생.

## ✅ 조치 및 개선 사항

현실 세계 반영 및 모델 학습 유효성 확보를 위한 코드 수정 완료.

* **시뮬레이션 파라미터 현실화 (`sumo/data_generator.py`)**
  * **용량 증대:** 50~100 kWh (현실 EV 수준)
  * **충전 목표:** 80~100% 설정 (충전 체류 시간 확보)
  * **초기 상태:** 출발 시 잔량을 10~40%로 부여하여 시뮬레이션 내 수요 발생 유도
* **파서 로직 수정 (`sumo/parse_outputs.py`)**
  * **하드코딩 제거:** `charging_output.xml`에서 각 차량의 실제 `actualBatteryCapacity`와 `maximumBatteryCapacity`를 동적 추출.
  * **정확도 향상:** 완벽히 일치하는 도착 시점 SOC 피처 추출 가능.

## 🚀 향후 계획 (Phase 3 진입)

* **백그라운드 작업 진행 중**
  * 개선된 파라미터를 적용하여 **162개 조합**의 배치 시뮬레이션 전면 재수행 중.
* **학습 파이프라인 대기 (`sumo/train_demand.py`)**
