# 프레젠테이션 기획안: SUMO와 Graph WaveNet을 활용한 전기차 충전 수요 예측 모델

## 프레젠테이션 개요

### 1. 청중 분석 (Audience Analysis)
*   **청중:** 모빌리티 AI 연구원, 도시 계획가, 스마트 그리드 엔지니어, 데이터 사이언티스트 (중급/고급)
*   **관심사:** 전기차(EV) 인프라 최적화, 시공간(Spatiotemporal) 데이터 모델링, 트래픽 시뮬레이션 활용법
*   **목표:** 합성 데이터 생성의 한계와 극복 방법, GNN(Graph Neural Network)을 통한 공간적 수요 예측의 실효성 이해

### 2. 프레젠테이션 목표 (Presentation Goals)
*   **핵심 목표 (Primary):** 복잡한 도심 네트워크에서 EV 충전 수요를 예측하기 위해 SUMO 시뮬레이션 데이터와 Graph WaveNet 아키텍처를 결합하는 파이프라인의 가치와 구현 방법을 전달한다.
*   **부가 목표 (Secondary):**
    *   단순 시계열(LSTM) 대비 시공간 모델(Graph WaveNet)의 우수성 이해
    *   물리적으로 유효한(Physically Grounded) 시뮬레이션 데이터 생성의 중요성 인지
    *   연구의 다음 단계(BMW 실데이터 적용 등) 비전 공유

## 본론 1: 기반 기술 및 데이터 생성 (10분)

### Slide 4: Microscopic Traffic Simulation (SUMO)
*   **내용:**
    *   왜 SUMO인가? (개별 차량 수준의 미시적 추적)
    *   가상의 베를린, 수천 대의 EV 에이전트
*   **시각 자료:** 베를린 OSM 기반 도로망과 그 위를 움직이는 노드(차량) 애니메이션
*   **발표자 노트:** "우리는 실제 데이터가 없다면, 완벽한 물리 법칙이 지배하는 가상 세계를 만들기로 했습니다. SUMO를 통해 베를린 시내를 달리는 수천 대의 EV를 시뮬레이션했습니다."

### Slide 5: Deep Dive - 시뮬레이션 파라미터의 함정
*   **내용:**
    *   단순한 장난감 차? No, 현실적인 EV 배터리 물리!
    *   [Before] 배터리: 5kWh (자전거 수준) -> 대기 시간 0
    *   [After] 배터리: 50~100kWh (현실 수준), 초기 잔량 10~40% 세팅
*   **시각 자료:** Before(비현실적)와 After(현실적 배터리)의 충전 시간 차이를 보여주는 비교 그래프
*   **발표자 노트:** "처음엔 치명적인 실수가 있었습니다. 시뮬레이션 속 차량 배터리를 5kWh로 설정해 충전이 1분 만에 끝났죠. 현실적인 병목 현상을 만들기 위해 100kWh급 배터리와 낮은 초기 잔량 부여라는 물리적 트릭이 필요했습니다."

### Slide 6: 데이터 파싱 (XML to Tensor)
*   **내용:**
    *   수 기가바이트의 이벤트를 텐서로 압축
    *   도착 시 잔여 배터리(SOC) 동적 추출 기술
*   **시각 자료:** 복잡한 XML 코드가 깔끔한 Numpy Tensor 큐브 (N_stations, T_bins, Features)로 변환되는 일러스트
*   **발표자 노트:** "모델은 XML을 읽지 못합니다. lxml iterparse를 활용해 메모리 누수 없이 충전 이벤트를 5분 단위의 시공간 텐서로 변환하는 정제 파이프라인을 구축했습니다."

## 본론 2: 시공간 모델 적용 (10분)

### Slide 7: 단순 시계열의 한계 (Transition)
*   **내용:** "LSTM은 시간을 알지만, 공간을 모릅니다."
*   **시각 자료:** A 충전소가 꽉 차면 B 충전소로 이동하는 '풍선 효과'를 보여주는 다이어그램

### Slide 8: Graph WaveNet 아키텍처
*   **내용:**
    *   시간 캡처: 1D Dilated Convolutions (빠르고 넓은 수용장)
    *   공간 캡처: Graph Convolutional Networks (GCN)
*   **시각 자료:** Graph WaveNet의 핵심 블록 다이어그램 (TCN 블록과 GCN 블록이 결합된 형태)
*   **발표자 노트:** "충전 수요는 거리에 반비례합니다. Graph WaveNet은 시간적 흐름뿐만 아니라 도로망이라는 '그래프(Graph)' 구조를 동시에 학습합니다."

### Slide 9: 핵심 무기 - Adaptive Adjacency Matrix
*   **내용:**
    *   물리적 거리의 한계 극복
    *   데이터가 스스로 찾아내는 숨겨진 상관관계 (고속도로 연결 등)
*   **시각 자료:** 노드 간의 물리적 거리(파란선)와 모델이 학습한 숨겨진 가중치(빨간선)의 비교 시각화
*   **발표자 노트:** "가장 강력한 점은 '적응형 인접 행렬'입니다. 모델 스스로 물리적으로 멀어 보여도 고속도로로 묶인 두 충전소 간의 밀접한 연관성을 찾아냅니다."

### Slide 10: 현재 학습 현황 (Status)
*   **내용:**
    *   162개 대규모 시나리오 데이터 추출 완료
    *   학습 파이프라인 백그라운드 구동 중

## 발표자 유의사항 (Speaker Guidelines)
*   **톤 앤 매너:** 학술적이지만 실용주의적인 태도. 실패 경험(배터리 용량 세팅 오류)을 솔직히 공유하며 청중과의 공감대 형성.
*   **수식 배제:** Graph WaveNet의 수식은 가급적 배제하고 직관적인 '공간의 연결성'을 강조.
*   **시간 배분:** 데이터 파이프라인의 어려움(본론 1)에 충분한 시간을 할애하여, 양질의 데이터가 모델보다 중요하다는 AI 연구의 본질을 역설할 것.
